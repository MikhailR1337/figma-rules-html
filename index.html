Figma Design Rules

Essential principles to create better, more maintainable designs in Figma. Follow these rules to improve your design workflow and collaboration.

1

Grouping Appropriate Layers Together

Organize related elements into logical groups to maintain a clean layer structure. This makes your design easier to navigate, edit, and collaborate on.

✅ Good Examples:

  • Group all navigation elements under "Navigation"
  • Combine form fields and labels in "Form Elements"
  • Keep all icons together in "Icons" group

❌ Avoid:

  • Scattered layers without logical grouping
  • Mixing UI elements with content layers
2

Applying Auto Layout

Use Auto Layout to create responsive, flexible components that automatically adjust to content changes and maintain consistent spacing.

✅ Best Practices:

  • Use for navigation menus, button groups, and card layouts
  • Set appropriate padding and spacing values
  • Choose horizontal or vertical alignment based on content

3

Defining Gaps, Alignments & Constraints

Establish consistent spacing, proper alignment, and responsive constraints to ensure your design scales properly across different screen sizes.

Gaps

Use consistent spacing values (8px, 16px, 24px) between elements for visual harmony.

Alignments

Align elements to create visual order and improve readability.

Constraints

Set proper constraints for responsive behavior and component flexibility.

💡 Pro Tips:

  • • Use 8px grid system for consistent spacing
  • • Align text baselines for better typography
  • • Set left/right constraints for horizontal scaling
  • • Use top/bottom constraints for vertical scaling
4

Handling Text Layers Properly

Proper text layer management ensures consistency, accessibility, and easier content updates across your design system.

✅ Text Best Practices:

  • Use text styles for consistent typography
  • Set proper line height and letter spacing
  • Enable text auto-resize for dynamic content
  • Use semantic text layers (H1, H2, Body, Caption)

Using Background Images

Properly implement background images to enhance visual appeal while maintaining performance and accessibility standards.

✅ Implementation Guidelines:

  • Use appropriate fill modes (Fill, Fit, Crop, Tile)
  • Set proper image constraints and scaling
  • Ensure text remains readable over backgrounds
  • Optimize image file sizes for performance

6

Using Strokes Instead of Lines for Borders

Apply strokes to shapes and frames instead of using line elements for borders. This approach provides better control and consistency in your designs.

✅ Why Use Strokes:

  • Better control over border radius and styling
  • Easier to maintain and update consistently
  • More flexible for responsive design
  • Better integration with design systems

7

Contextually Naming Layers

Use descriptive, semantic names for your layers to improve collaboration, maintainability, and overall design workflow efficiency.

✅ Naming Conventions:

  • Use descriptive names: "Primary Button" instead of "Rectangle 1"
  • Include state information: "Button/Hover", "Button/Pressed"
  • Use consistent prefixes: "Icon/", "Text/", "Container/"
  • Group related elements: "Navigation/Logo", "Navigation/Menu"

✅ Good Names

• Header/Navigation
• Button/Primary/Default
• Card/Product/Image
• Icon/Arrow/Right
• Text/Heading/H1

❌ Avoid

• Rectangle 1
• Group 2
• Copy
• Untitled
• Frame 1

Follow these rules to create better, more maintainable Figma designs. Happy designing! 🎨
